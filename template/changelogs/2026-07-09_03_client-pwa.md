# Client PWA (manifest + offline-first service worker)

**Date:** 2026-07-09
**Type:** Feature

## Summary

Added Progressive Web App support to the Next.js 15 client: a typed
`app/manifest.ts` (served at `/manifest.webmanifest`, auto-linked by Next), an
offline-first `public/sw.js`, an `/offline` fallback route, and a production-only
service-worker registrar mounted in the root layout. Client build compiles clean
and emits `/manifest.webmanifest` + `/offline`; 38 client tests pass.

## Changes

- `client/src/app/manifest.ts`: typed `MetadataRoute.Manifest` (name, icons,
  standalone display, theme/background colors).
- `client/public/sw.js`: install precaches the public app shell (`/`, `/offline`);
  activate cleans stale caches; fetch is network-first for navigations (cached →
  `/offline` fallback) and cache-first for same-origin static assets.
- `client/public/icon.svg`: SVG app icon (`sizes: "any"`) — no binary PNGs.
- `client/src/app/offline/page.tsx`: offline fallback page.
- `client/src/components/common/ServiceWorkerRegistrar.tsx`: `'use client'`
  registrar that registers `/sw.js` **only in production** (dev HMR conflicts with
  SW caching); renders nothing.
- `client/src/app/layout.tsx`: mounts the registrar; adds `viewport.themeColor`.

## Gotchas captured

- Register the SW only in production.
- Precache only PUBLIC routes — precaching a protected/redirecting route makes
  `cache.addAll` reject and the install fail.
- `themeColor` belongs in the `viewport` export (Next 14+), not `metadata`.
